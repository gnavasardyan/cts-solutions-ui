Техническое задание на разработку пользовательского интерфейса системы прослеживаемости элементов металлоконструкций

1. Введение
1.1. Цель документа
Настоящее ТЗ определяет требования к пользовательскому интерфейсу (UI) системы цифровой прослеживаемости металлоконструкций, обеспечивающей контроль перемещения изделий от завода-производителя до точки использования.
1.2. Назначение системы
Система предназначена для:
Формирования уникальных идентификаторов (кодов) на металлоконструкциях
Отслеживания перемещения изделий через 3 контрольные точки (завод-производитель, площадка хранения, площадка использования)
Фиксации изменений статусов изделий в реальном времени
Обеспечения прозрачности логистических операций для всех участников процесса
1.3. Объект разработки
Пользовательский интерфейс веб-приложения, включающий:
Административную панель управления
Интерфейс для нанесения маркировки на заводе-производителе
Интерфейс для сканирования и контроля на промежуточных точках
Мобильное приложение для работы на точках контроля
2. Цели и задачи UI
2.1. Основные цели
Обеспечение интуитивно понятного взаимодействия для персонала с разным уровнем цифровой грамотности
Минимизация ошибок при регистрации перемещений изделий
Обеспечение работы в условиях производственной среды (освещение, перчатки, вибрация)
Поддержка всех этапов процесса прослеживаемости в единой системе
2.2. Решаемые задачи
Визуализация текущего статуса всех отслеживаемых элементов
Упрощение процесса нанесения и считывания маркировки
Предоставление инструментов для оперативного реагирования на отклонения
Формирование отчетности в формате, удобном для анализа
3. Требования к пользовательскому интерфейсу
3.1. Общие требования
Язык интерфейса: русский (возможность добавления английского в будущем)
Поддержка работы в offline-режиме с последующей синхронизацией данных
Адаптивный дизайн для отображения на экранах от 5" (мобильные устройства) до 24" (стационарные рабочие места)
Минимальная поддержка браузеров: Chrome, Firefox, Edge (последние 2 версии)
3.2. Функциональные требования
3.2.1. Модуль аутентификации и управления ролями
Единый вход для всех участников системы
Роли пользователей:
Администратор системы
Оператор завода-производителя
Кладовщик (площадка хранения)
Мастер участка (площадка использования)
Аудитор (только просмотр)
Гибкая настройка прав доступа к функциям системы
Возможность временного доступа для подрядчиков
3.2.2. Панель управления (Dashboard)
Отображение в реальном времени:
Общее количество отслеживаемых элементов
Распределение по статусам (производство, в пути, на хранении, в монтаже)
Карта перемещения с указанием текущего местоположения
Предупреждения о задержках или отклонениях
Фильтрация данных по:
Дате производства
Марке металлоконструкции
Пунктам назначения
Статусу перемещения
3.2.3. Модуль маркировки (завод-производитель)
Интерфейс для генерации уникального кода:
Автоматическая привязка к данным изделия (чертеж, партия, ГОСТ)
Предварительный просмотр кода (DataMatrix)
Контроль качества нанесения (интеграция с камерой)
Форма заполнения с подтверждением данных:
Выбор типа металлоконструкции из справочника
Указание габаритов и веса
Прикрепление фото готового изделия
Подтверждение оператором
3.2.4. Модуль контроля на точках перемещения
Интерфейс сканирования:
Режим "быстрого сканирования" с автоматическим переходом к следующему элементу
Визуальная и звуковая индикация успешного считывания
Отображение текущего статуса элемента при сканировании
Форма фиксации перемещения:
Автоопределение точки контроля по геолокации
Выбор операции (приемка, отправка, инвентаризация)
Возможность добавления комментариев и фото
3.2.5. Модуль просмотра истории и отчетности
Детализированная карта перемещения для каждого элемента:
Временные метки всех операций
Фотофиксация на каждом этапе
Информация об операторах
Генерация отчетов:
Сводные ведомости по перемещениям
Отчеты по задержкам
Акт приема-передачи в формате PDF
Экспорт данных в Excel
3.3. Нефункциональные требования
Время отклика системы: не более 2 секунд при стандартной нагрузке
Поддержка одновременной работы не менее 50 пользователей
Устойчивость к ошибкам ввода (защита от случайных действий)
Минимальные требования к оборудованию:
Мобильные устройства: Android 8.0+, iOS 12+
Стационарные рабочие места: 4 ГБ ОЗУ, FullHD дисплей
Обучаемость: время освоения базовых функций не более 30 минут
3.4. Требования к интеграции с оборудованием
Поддержка промышленных сканеров штрихкодов (Zebra, Honeywell)
Интеграция с принтерами для нанесения маркировки (SATO, Brother)
Поддержка работы с RFID-метками (при наличии оборудования)
Использование встроенного сканера мобильных устройств
Геолокация через GPS/ГЛОНАСС для фиксации точек контроля
4. Основные сценарии использования
4.1. Нанесение маркировки на заводе
Оператор авторизуется в системе
Система определяет текущий производственный заказ
Генерируется уникальный код с привязкой к данным изделия
Код наносится на металлоконструкцию
Фиксируется фото готового изделия с маркировкой
Статус элемента меняется на "Готов к отправке"
4.2. Приемка на площадке хранения
Кладовщик сканирует код прибытия
Система отображает ожидаемые элементы по накладной
При несоответствии – возможность пометить расхождение
Фиксация времени приемки и подписи ответственного
Автоматическое обновление статуса в центральной системе
4.3. Контроль на площадке использования
Мастер сканирует код перед монтажом
Система проверяет соответствие проектной документации
Фиксируется факт ввода в эксплуатацию
Формирование акта ввода в эксплуатацию
Обновление статуса элемента до "В эксплуатации"
5. Этапы разработки и приемки
5.1. Этапы работ
Разработка UI:
Верстка адаптивного интерфейса
Интеграция с backend-системой
Тестирование на реальном оборудовании
Тестирование и адаптация:
Проверка в условиях, приближенных к производственным
Корректировка интерфейса на основе feedback'а
Обучение ключевых пользователей
5.2. Критерии приемки
Все основные сценарии использования реализованы без критических ошибок
Время выполнения базовых операций не превышает 15 секунд
Успешное прохождение тестирования с персоналом заказчика
Наличие документации по использованию интерфейса
Соответствие требованиям эргономики для производственной среды
6. Дополнительные требования
Все текстовые элементы должны соответствовать ГОСТ 12.0.004-2015
Интерфейс должен поддерживать работу в перчатках (минимальный размер элементов управления – 48x48 px)
Цветовая схема должна обеспечивать контрастность не менее 4.5:1 для соответствия WCAG 2.0
Обязательное наличие подсказок и туториалов для первых 3 сессий работы пользователя
